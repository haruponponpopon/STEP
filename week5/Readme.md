## 配列を削除する機能  
例えば3番目以降を削除するなら  
```
#include <iostream>
#include <vector>
#include <algorithm>

int main(){
	std::vector<int> array = {1,2,3,4,5};
	array.erase(std::remove_if(array.begin(), array.end(), [](int i) {return i>=3; }), array.end());
	for (int i=0; i<(int)array.size(); i++){
		std::cout << array.at(i) << " ";
	}
	std::cout << std::endl;
}
```

## mapにconstをつけて関数の引数としたいとき  
findとiter->secondを組み合わせるとconstをつけてもエラーにならない。予期せぬ書き換えを防ぐためにも、積極的に使っていこう。
```
#include <iostream>
#include <map>
#include <string>

void search_key (const std::map<std::string, int>& fruit_map, std::string key){
	auto iter = fruit_map.find(key);
	if (iter == fruit_map.end()) {
		std::cout << "not exist" << std::endl;
  		return;  // 見つからなかった
	}
	std::cout << iter->second<< std::endl;
}

int main(){
	std::map<std::string, int> fruit_map;
	fruit_map["apple"] = 1;
	fruit_map["banana"] = 2;
	search_key(fruit_map, "banana");
	search_key(fruit_map, "orange");
}
```

### long double VS double  
どちらも8byte型なのでメモリに関しては考えなくてよさそうでした。  
#### doubleの良いところ  
long doubleより文字数が少ないので、コードが冗長になりにくい。  
#### long doubleの良いところ  
doubleよりも正確  
例えば`input_0.csv`ファイルの最後の行を小数点以下20桁でprintすると  
`150.17533883877582,   25.512728869805677`元のデータ  
`150.17533883877581502,25.512728869805677334`double  
`150.17533883877582,   25.512728869805676999`long double  
とlong doubleの方が元のデータに近い値となる。  
#### 今回選んだのは  
double型を選びました。元のデータの桁数を考慮するとdouble型でも十分表せると考えたためです。

## 説明  
### read_file関数  
input_x.csvファイルを読み込んで配列に入れて返します。引数にxの部分を取ります。ファイルに予期せぬことが書かれていた場合、適宜エラーを返します。  
### city_distance関数  
二都市のx,y座標をそれぞれ受け取り、距離を計算して返します。  
### calc_total_distance関数  
全ての都市の座標が入った配列と都市の訪問順序を受け取り、その経路でスタートの都市から再びスタートの都市に戻ってくるまでの距離を計算して返します。
### choose_nearest_city1関数  
ただの貪欲法です。表の二列目から四列目までの実行結果はこちらの関数で得られます。city_order配列に現在の都市から一番近く、まだ訪れていない都市を順番に入れていきます。一番最初の都市は0となっています。得られた経路を返します。
### choose_nearest_city2関数  
工夫された貪欲法です。(for文で回しているだけですが、、、)表の五列目の実行結果はこちらの関数で得られます。main関数内の呼び出し時に`choose_nearest_city1関数`の`1`の部分を`2`に変えてください。スタートの都市をfor文で0からN-1まで試して貪欲法で経路を決め、もっとも総距離が短くなった経路を返します。
### cross_city関数  
四つの都市を受け取り、都市A1と都市A2、都市B1と都市B2の経路が交差しているか否かを判定します。 
f(x,y) = (x1-x2)(y-y1)+(x1-x)(y1-y2)
g(x,y) = (x1-x2)(y-y1)+(x1-x)(y1-y2) を考え、f(x,y)とg(x,y)の正負が異なるときfとgに入れた点同士は点(x1,y1),(x2,y2)を通る直線の反対側にいることになります。`judge1=f(x,y)*g(x,y)  judge2=f(x,y)*g(x,y)`であり、judge1と2の違いは点となる都市と直線となる都市が入れ替わっています。(説明がうまくできずにすいません)
```
* /
 /
/  *  このときf(x,y)*g(x,y)>0になる。*の都市と/の都市を入れ替える。両方ともf(x,y)*g(x,y)>0となれば交差する。
```

### two_opt関数  
まずstartの都市と最後の都市を見て交差していたら交差を解きます。その後二重ループで全ての隣接する都市の組み合わせについて交差していたら解くことを繰り返します。表の四番目にj=0との記述がありますが、交差を解いたら新しい交差ができてしまい貪欲法のみを用いた時より出力される経路が長くなってしまうバグが生じたので交差したら再び最初から交差がないかどうか見ていきます。しかしこの方法でも都市の数が大きかった時に無限ループが生じてしまったのでループの上限を設けました。そうしたら表にある通り、都市の数によってはうまくいかないことがありました。(原因は考えましたがこれ以上は難しいので提出させていただきました。)
